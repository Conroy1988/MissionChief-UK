# Contributing

Contributions should improve accuracy, coverage or usability without weakening the evidence standard.

## Useful contributions

- Reproduced corrections to game values or behaviour
- Missing aliases and UK terminology
- Screenshots that demonstrate a mechanic clearly
- New-player and advanced strategy improvements
- Script installation, configuration and compatibility documentation
- Accessibility, mobile and iOS guidance
- Structured data records with sources and verification dates

## Before submitting

1. Search the repository and open issues for existing work.
2. Confirm that the information applies to the UK version.
3. Separate verified facts from recommendations.
4. Record how and when a claim was checked.
5. Avoid copying substantial third-party text or assets.
6. Link related guides and reference records.

## Guide front matter

Substantive pages should use metadata similar to:

```yaml
---
title: Example guide
status: review_required
last_verified: 2026-07-22
tags:
  - fire
  - vehicles
---
```

## Writing rules

- Use current UK game names first.
- Define abbreviations on first use.
- State prerequisites before instructions.
- Use exact values only when evidence supports them.
- Explain exceptions and cross-service dependencies.
- Avoid presenting one play style as universally optimal.
- Do not conceal uncertainty.

## Corrections

A correction should include:

- the page or record affected;
- the current claim;
- the proposed replacement;
- evidence or reproduction steps;
- the date checked;
- any account conditions that may affect the result.

## Third-party scripts

Tool documentation must link to the maintainer-controlled source, explain requested permissions, identify supported devices and state whether the tool is actively maintained. Do not redistribute modified third-party code without the relevant licence and attribution.
