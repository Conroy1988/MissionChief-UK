# MissionChief UK v1.0.1

**Released:** 22 July 2026  
**Release type:** Quality and validation patch  
**Static Data API:** v1, backwards compatible

MissionChief UK v1.0.1 hardens the production guide, browser tools and release process established in v1.0.0. It does not change the canonical mission, resource, infrastructure or qualification counts.

## Production baseline

```text
62 verified mission records
46 canonical deployable-resource records
18 canonical infrastructure records
11 qualification records
11 represented operational service groups
13 verified mission-data batches
Static Data API v1.0.1
```

## Added

- Playwright acceptance testing for Chromium and Firefox desktop browsers.
- WebKit acceptance profiles for iPhone and iPad layouts.
- Functional coverage for Mission Lookup, Resource Comparison, Qualification Comparison, Concurrent Fleet Planner and Query Catalogue.
- MkDocs Material instant-navigation lifecycle testing.
- Public API consistency testing across all eight v1 endpoints.
- Critical WCAG A/AA scanning with `@axe-core/playwright`.
- Responsive-layout checks for page-level horizontal overflow.
- First-party HTTP, request and JavaScript runtime failure detection.
- Retained Playwright reports, traces, screenshots and videos for failed Actions runs.
- A deterministic internal documentation-link and heading-anchor audit.

## Changed

- Validation now performs inexpensive data, export, link and strict-build checks before installing browser binaries.
- Pages deployment now requires a successful cross-browser acceptance matrix after deployment.
- Release readiness now protects the Playwright configuration, test suite, QA guide and link auditor as release-critical assets.
- Browser, smoke and release checks now derive the expected version from `data/version.json` rather than embedding `1.0.0`.
- The release workflow now supports future compatible v1 semantic-version releases and rejects tags that point at a different commit.
- The repository README and Quality Assurance guide now reflect the complete production validation pipeline.

## API compatibility

The API remains under:

```text
assets/data/v1/
```

The collection envelopes, endpoint paths, canonical IDs and evidence semantics are unchanged. `data_version` advances to `1.0.1` to identify this validated publication.

## Data impact

No canonical records were added, removed or reclassified in this patch. The verified collection counts remain unchanged from v1.0.0.

Unavailable Recovery and Bomb Disposal response pages were rechecked during preparation. Their missing response tables remain intentionally unpopulated because suitable primary evidence was still unavailable.

## Validation sequence

The release is eligible for publication only after:

1. structured-data schema and relationship validation;
2. deterministic export and FAQ generation;
3. repository and API readiness auditing;
4. internal link and heading-anchor auditing;
5. strict MkDocs build and built-site auditing;
6. Chromium acceptance against the built site;
7. GitHub Pages deployment;
8. deployed HTTP and API smoke testing;
9. Chromium, Firefox, iPhone WebKit and iPad WebKit acceptance.
