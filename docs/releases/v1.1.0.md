# MissionChief UK v1.1.0

**Released:** 23 July 2026  
**Release type:** Mission intelligence expansion  
**Static Data API:** v1, backwards compatible

MissionChief UK v1.1.0 expands Mission Lookup from the original 62 canonical records to the complete public United Kingdom mission catalogue, begins the enforced programme to reach 100% fully canonical coverage and preserves a strict distinction between official source data and normalized operational evidence.


## Accuracy controls

- Official values are preserved verbatim.
- Unknown internal keys remain visible and are never guessed.
- Canonical records remain the higher-trust dispatch model.
- Seasonal availability, base missions, overlays and variants remain distinct.
- New official fields are retained automatically.
- Duplicate IDs, missing names, stale reconciliation, stale inventories and source/public mismatches fail validation.
- A mission cannot enter a mapped gate while any official requirement, chance or prerequisite key is unclassified.
- `not-applicable` mappings require narrow allowed values.
- Strict missions must exactly match mapped official quantities, probabilities and canonical preconditions.
- Aggregate diagnostics report every promoted identity or mapping failure in one run.
- Unresolved relationship IDs block fully canonical promotion.
- Self-referential base identities remain distinct from genuine variants.

## Automated maintenance

The official refresh runs daily and may be dispatched manually. An unchanged source SHA and unchanged verification state produce no repository change. Changed data must pass importer validation, publication, compaction, coverage reconciliation, registry merge, aggregate identity diagnostics, key mapping validation, candidate and key-backlog analysis, and catalogue auditing before commit and deployment.
