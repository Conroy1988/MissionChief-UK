# MissionChief UK v1.0.0

**Released:** 22 July 2026  
**Core programme:** Stages 1–34 complete  
**Static Data API:** v1

MissionChief UK v1.0.0 is the first production release of the independent, evidence-led UK command knowledgebase.

## Production baseline

```text
62 verified mission records
46 canonical deployable-resource records
18 canonical infrastructure records
11 qualification records
11 represented operational service groups
13 verified mission-data batches
Static Data API v1.0.0
```

## Major capabilities

- searchable MkDocs Material documentation deployed through GitHub Pages;
- canonical mission, vehicle, infrastructure and qualification datasets;
- Draft 2020-12 schema validation and cross-record referential integrity;
- guaranteed, probabilistic, conditional and alternative requirement modelling;
- explicit mission variants, overlays, patients, prisoners, personnel ranges and towing outcomes;
- Railway Police, railway fire response, Bomb Disposal, Airfield Operations and Recovery coverage;
- generated public JSON collections, manifest, search index, FAQ and OpenAPI 3.1 contract;
- browser-side mission lookup, resource comparison, qualification comparison, fleet planning and evidence search;
- release-readiness, built-site and deployed-Pages smoke testing.

## Public API

The read-only API is published under `assets/data/v1/` and includes:

```text
manifest.json
missions.json
vehicles.json
infrastructure.json
training.json
search-index.json
faq.json
openapi.json
```

The v1 path is stable for the lifetime of the major version. Additive compatible fields may be introduced without changing the path; breaking changes require a new versioned path.

## Evidence policy

`verified` applies only to populated fields. An omitted value is unknown, not zero.

Some records use directory-level evidence because an individual UK mission response table was unavailable. Such records preserve verified IDs, rewards, POIs and generation preconditions without inventing dispatch requirements.

Alternative groups express a total quantity from any combination of qualifying resources. Conditional and probabilistic requirements remain separate. Recovery towing outcomes remain separate from emergency dispatch requirements.

## Known evidence gaps

The release intentionally leaves several areas for continuous evidence maintenance:

- additional vehicle prices, crew limits and exact training durations;
- unavailable Bomb Disposal and Recovery response tables;
- infrastructure costs, capacities and compatible parent buildings;
- dispatch-allocation behaviour where one vehicle might satisfy overlapping alternative groups;
- new UK missions and balancing changes released after 22 July 2026.

These are evidence-enrichment tasks, not unfinished core architecture.

## Validation and deployment

The v1.0.0 release is created only after:

1. structured-data schema validation;
2. mission-to-resource and mission-to-infrastructure checks;
3. deterministic export and FAQ generation;
4. repository/API release-readiness auditing;
5. JavaScript syntax validation;
6. strict MkDocs build;
7. built-site output auditing;
8. GitHub Pages deployment;
9. deployed-site and public-API smoke testing.
