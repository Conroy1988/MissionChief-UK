# Reference Database

The reference layer will hold concise, searchable records linked to the explanatory guides.

## Record families

- Missions
- Vehicles and aliases
- Buildings and extensions
- Personnel capacities
- Training courses and qualifications
- Hospitals, prisons and destination capacities
- Costs, unlocks and prerequisites
- Script and tool compatibility

## Design principles

- Stable machine-readable identifiers
- UK display names plus known aliases
- Explicit units and data types
- Source and verification metadata
- Relationships between missions, resources and qualifications
- Human-readable generated indexes
- Validation before publication

See the [data standard](data-standard.md).
