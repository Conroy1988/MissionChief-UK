# Glossary and Search Aliases

MissionChief terminology varies between the game interface, emergency-service practice, community shorthand and third-party tools. The glossary provides one searchable translation layer.

## Alias rules

- Preserve the canonical UK game label as the primary term.
- Record common abbreviations and historic labels as aliases.
- Do not silently replace a current full name with community shorthand.
- Mark service-specific or tool-specific terminology.
- Record potentially ambiguous aliases with explanatory notes.
- Keep aliases separate from capabilities: two vehicles sharing an informal label are not automatically interchangeable.

## Planned groups

- vehicle abbreviations;
- station and extension terminology;
- personnel and training names;
- mission-interface terminology;
- alliance terminology;
- LSSM module names;
- userscript and browser-tool terminology;
- maritime and rescue nomenclature.

## Search behaviour

Aliases should improve discovery without changing displayed canonical names. Structured records should therefore store aliases explicitly so the documentation search, future API and planning tools can use the same vocabulary.
